# 常见排序算法

## 一、总览

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  算法    │ 最好     │ 平均     │ 最坏     │ 空间     │ 稳定性   │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ 冒泡     │ O(n)     │ O(n²)    │ O(n²)    │ O(1)     │ ✅ 稳定  │
│ 选择     │ O(n²)    │ O(n²)    │ O(n²)    │ O(1)     │ ❌ 不稳定 │
│ 插入     │ O(n)     │ O(n²)    │ O(n²)    │ O(1)     │ ✅ 稳定  │
│ 希尔     │ O(n log n)│O(n^1.3) │ O(n²)    │ O(1)     │ ❌ 不稳定 │
│ 归并     │ O(n log n)│O(n log n)│O(n log n)│ O(n)    │ ✅ 稳定  │
│ 快速     │ O(n log n)│O(n log n)│ O(n²)    │ O(log n) │ ❌ 不稳定 │
│ 堆排序   │ O(n log n)│O(n log n)│O(n log n)│ O(1)    │ ❌ 不稳定 │
│ 计数     │ O(n+k)    │ O(n+k)   │ O(n+k)   │ O(n+k)  │ ✅ 稳定  │
│ 桶排序   │ O(n+k)    │ O(n+k)   │ O(n²)    │ O(n+k)  │ ✅ 稳定  │
│ 基数     │ O(nk)     │ O(nk)    │ O(nk)    │ O(n+k)  │ ✅ 稳定  │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

**一句话选算法：**
- 数据量小（<50）或基本有序 → **插入排序**
- 需要稳定 + O(n log n) → **归并排序**
- 实际工程中最快 → **快速排序**
- 只关心 Top K → **堆排序**
- 整数 + 范围有限 → **计数排序**

## 二、冒泡排序

```
  思路：相邻两两比较，大的往后沉
  每轮把最大的元素冒泡到最后

  [5, 3, 8, 1]
   5↔3 → [3, 5, 8, 1]
     5↔8 → [3, 5, 8, 1]  (不换)
       8↔1 → [3, 5, 1, 8]  第1轮结束，8归位
  ──────────
  [3, 5, 1, 8]
   3↔5 → [3, 5, 1, 8]
     5↔1 → [3, 1, 5, 8]    第2轮结束，5归位
  ...
```

```java
public void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;  // 优化：本轮没有交换则已有序
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr, j, j + 1);
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```

## 三、选择排序

```
  思路：每轮找到剩余部分的最小值，放到已排序末尾
  不稳定：交换可能破坏相等元素的相对顺序

  [5, 3, 8, 1]
  找最小(1) → [1, 3, 8, 5]  交换 5 和 1
  找最小(3) → [1, 3, 8, 5]  不变
  找最小(5) → [1, 3, 5, 8]  交换 8 和 5
```

```java
public void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        if (minIdx != i) {
            swap(arr, i, minIdx);
        }
    }
}
```

## 四、插入排序

```
  思路：像打牌整理手牌，拿到一张插入到已排序部分的正确位置
  数据基本有序时接近 O(n)

  [5, 3, 8, 1]
   5   │ 3 ← 把3插到5前面 → [3, 5, 8, 1]
   3,5 │ 8  ← 8在正确位置 → [3, 5, 8, 1]
  3,5,8│ 1 ← 1插到最前面 → [1, 3, 5, 8]
```

```java
public void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        // 比 key 大的都往后移
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

## 五、希尔排序

```
  思路：插入排序的改进版，先大跨步排序，再逐步缩小跨步
  跨步序列：gap = n/2, n/4, ..., 1

  原始: [8, 3, 6, 1, 9, 2, 7, 4]
  gap=4: 比较 (8,9)(3,2)(6,7)(1,4) → [8,2,6,1,9,3,7,4]
  gap=2: 比较 (8,6,9,7)(2,1,3,4)   → [6,1,7,2,8,3,9,4]
  gap=1: 普通插入排序               → [1,2,3,4,6,7,8,9]
```

```java
public void shellSort(int[] arr) {
    int n = arr.length;
    for (int gap = n / 2; gap > 0; gap /= 2) {
        // 对每个 gap 做插入排序
        for (int i = gap; i < n; i++) {
            int key = arr[i];
            int j = i;
            while (j >= gap && arr[j - gap] > key) {
                arr[j] = arr[j - gap];
                j -= gap;
            }
            arr[j] = key;
        }
    }
}
```

## 六、归并排序

```
  思路：分治法，先递归拆成单个元素，再两两合并有序数组

  拆分阶段：                      合并阶段：
  [5,3,8,1]                      [1,3,5,8]
    ╱   ╲                            ╱   ╲
 [5,3] [8,1]                    [3,5] [1,8]
  ╱ ╲   ╱ ╲                      ╱ ╲   ╱ ╲
 [5][3][8][1]                  [5][3][8][1]

 合并核心：两个有序数组合并成一个有序数组
```

```java
public void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

private void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;

    // 两路归并
    while (i <= mid && j <= right) {
        temp[k++] = arr[i] <= arr[j] ? arr[i++] : arr[j++];
    }
    while (i <= mid)  temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    // 拷回原数组
    System.arraycopy(temp, 0, arr, left, temp.length);
}
```

## 七、快速排序

```
  思路：选一个 pivot，小的放左边，大的放右边，递归排左右
  核心操作：partition（分区）

  pivot = 4
  [5, 3, 8, 1, 4, 7]
  小的放左 大的放右
   [3, 1, 4, 5, 8, 7]
       │
    pivot归位
     ╱     ╲
  [3,1]  [5,8,7]
   ╱ ╲    ╱  ╲
  [1][3] [5][8,7]
              ╱ ╲
            [7][8]
```

```java
public void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int pivotIdx = partition(arr, left, right);
    quickSort(arr, left, pivotIdx - 1);
    quickSort(arr, pivotIdx + 1, right);
}

private int partition(int[] arr, int left, int right) {
    int pivot = arr[right];  // 选最右为基准
    int i = left;            // i 指向"小元素区的下一个位置"

    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            swap(arr, i, j);
            i++;
        }
    }
    swap(arr, i, right);  // pivot 归位
    return i;
}
```

### 优化技巧

```java
// 1. 三数取中选 pivot —— 避免最坏 O(n²)
int mid = left + (right - left) / 2;
if (arr[left] > arr[mid]) swap(arr, left, mid);
if (arr[left] > arr[right]) swap(arr, left, right);
if (arr[mid] > arr[right]) swap(arr, mid, right);
swap(arr, mid, right);  // 中间值放到 right 作为 pivot

// 2. 小数组用插入排序 —— 减少递归开销
if (right - left < 20) {
    insertionSort(arr, left, right);
    return;
}
```

## 八、堆排序

```
  思路：建大顶堆 → 堆顶(最大值) 换到末尾 → 调整剩余堆 → 重复

  [4, 10, 3, 5, 1]
  
  建堆:
      4              10            10
     / \     →      / \     →     / \
    10  3          5   3         5   4
   /              /             /
  5              4             3

  排序: 堆顶与末尾交换 → 调整
  [10, 5, 4, 3, 1] → 10 归位 → [5, 3, 1, 4] 调整 → ...
```

```java
public void heapSort(int[] arr) {
    int n = arr.length;
    // 1. 建大顶堆（从最后一个非叶子节点开始）
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    // 2. 逐个取堆顶
    for (int i = n - 1; i > 0; i--) {
        swap(arr, 0, i);        // 堆顶(最大)移到末尾
        heapify(arr, i, 0);     // 调整剩余元素
    }
}

private void heapify(int[] arr, int heapSize, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < heapSize && arr[left] > arr[largest]) largest = left;
    if (right < heapSize && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr, i, largest);
        heapify(arr, heapSize, largest);  // 递归调整子树
    }
}
```

## 九、计数排序

```
  思路：统计每个值出现次数，按顺序输出
  限制：只适合整数 + 值范围不大

  arr = [2, 5, 3, 0, 2, 3, 0, 3]
  
  count[0]=2, count[1]=0, count[2]=2, count[3]=3, count[4]=0, count[5]=1
  
  按 count 输出: 0,0, 2,2, 3,3,3, 5
```

```java
public void countingSort(int[] arr) {
    int max = Arrays.stream(arr).max().getAsInt();
    int min = Arrays.stream(arr).min().getAsInt();
    int range = max - min + 1;

    int[] count = new int[range];
    int[] output = new int[arr.length];

    // 计数
    for (int val : arr) {
        count[val - min]++;
    }
    // 前缀和（用于确定每个值在输出数组中的位置）
    for (int i = 1; i < range; i++) {
        count[i] += count[i - 1];
    }
    // 反向遍历保证稳定性
    for (int i = arr.length - 1; i >= 0; i--) {
        output[--count[arr[i] - min]] = arr[i];
    }
    System.arraycopy(output, 0, arr, 0, arr.length);
}
```

## 十一、面试题

**Q1: 哪些排序是稳定的？稳定和不稳定有什么区别？**

> 稳定：相等元素的相对顺序在排序后不变 → 冒泡、插入、归并、计数、桶、基数。
> 不稳定：相等元素可能被交换顺序 → 选择、希尔、快速、堆。
> 稳定的意义：比如先按成绩排序，再按姓名排序——如果排序不稳定，第二次排序会破坏第一次的结果。

**Q2: 为什么快速排序在实际中最快？**

> ① 常数因子小——partition 操作只有简单的比较和交换 ② 空间局部性好——顺序访问数组，缓存命中率高 ③ 归并需要 O(n) 额外数组，快速只需要 O(log n) 递归栈。但最坏 O(n²)，工程中用三数取中或随机 pivot 规避。

**Q3: Arrays.sort() 为什么对基本类型和对象类型用不同算法？**

> 基本类型（int[]）：用双轴快速排序——不计较稳定性，追求速度。
> 对象类型（String[]等）：用 TimSort（归并+插入混合）——对象排序需要稳定性，且对象的比较开销大，TimSort 利用"部分有序"特性减少比较次数。

**Q4: 1TB 数据需要排序，内存只有 1GB，怎么办？**

> 外部排序：① 把 1TB 切分成 1000 个 1GB 块，每块读入内存用快速排序排好，写回磁盘 ② 对 1000 个有序块做多路归并（用堆维护每块的当前最小值），输出到最终文件。归并阶段用堆，每次只读各块当前行，内存消耗 = 堆大小 + 缓冲区。
