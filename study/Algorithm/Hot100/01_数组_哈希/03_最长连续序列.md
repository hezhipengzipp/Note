# 最长连续序列

> 难度：Medium
> 链接：https://leetcode.cn/problems/longest-consecutive-sequence
> 专题：数组/哈希

## 题目描述

给定一个未排序的整数数组，找出最长连续序列的长度。要求时间复杂度 O(n)。

## 思路

一句话：用 HashSet 存所有数，只从序列起点（num-1 不存在）开始向后计数。

## 图解

```
nums = [100, 4, 200, 1, 3, 2]

HashSet = {1, 2, 3, 4, 100, 200}

遍历每个数，判断是否为序列起点：
100: 99 不在 set 中 -> 是起点, 向后数 100,101... -> 长度 1
4:   3  在 set 中 -> 不是起点, 跳过
200: 199 不在 set 中 -> 是起点 -> 长度 1
1:   0  不在 set 中 -> 是起点, 向后数 1,2,3,4  -> 长度 4 ✓
3:   2  在 set 中 -> 不是起点, 跳过
2:   1  在 set 中 -> 不是起点, 跳过

结果：4
```

## 代码

```java
class Solution {
    public int longestConsecutive(int[] nums) {
        HashSet<Integer> set = new HashSet<>();
        for (int num : nums) {
            set.add(num);
        }

        int maxLen = 0;
        for (int num : set) {
            // 只从序列起点开始计数
            if (!set.contains(num - 1)) {
                int cur = num;
                int len = 1;
                while (set.contains(cur + 1)) {
                    cur++;
                    len++;
                }
                maxLen = Math.max(maxLen, len);
            }
        }
        return maxLen;
    }
}
```

## 复杂度

- 时间：O(n) - 每个元素最多被访问两次（外层一次，内层 while 一次）
- 空间：O(n) - HashSet 存储

## 易错点

1. **起点判断是关键**：`!set.contains(num - 1)` 确保每个序列只从头遍历一次，否则时间退化为 O(n²)
2. **遍历 set 不是 nums**：避免重复元素导致重复计算
3. **空数组返回 0**：边界条件
