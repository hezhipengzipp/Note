# 大文件多线程下载与线程池设计

## 一、多线程下载原理

```
  一个大文件 (100MB)

  1. 服务端支持 HTTP Range 请求
  2. 客户端切分文件为 N 块
  3. N 个线程分别下载各自的块
  4. 下载完成后按顺序合并

  ┌─────────────────────────────────────────────────────┐
  │                    100MB 文件                        │
  └─────────────────────────────────────────────────────┘
         │              │              │
         ▼              ▼              ▼
  ┌────────────┐ ┌────────────┐ ┌────────────┐
  │ 线程1       │ │ 线程2       │ │ 线程3       │
  │ 0-25MB     │ │ 25-50MB    │ │ 50-100MB   │
  │ Range:     │ │ Range:     │ │ Range:     │
  │ bytes=0-   │ │ bytes=     │ │ bytes=     │
  │  26214400  │ │ 26214401-  │ │ 52428801-  │
  │            │ │ 52428800   │ │            │
  └─────┬──────┘ └─────┬──────┘ └─────┬──────┘
        │              │              │
        ▼              ▼              ▼
  ┌─────────────────────────────────────────────┐
  │  写入同一个文件的不同偏移位置 (RandomAccessFile)  │
  └─────────────────────────────────────────────┘
```

## 二、线程池参数怎么定

### 2.1 核心公式

```
  线程数 = 入参可控，但有几个约束：

  1. 线程数 ≤ 服务端最大连接数限制（避免被 ban）
  2. 线程数 ≤ 客户端带宽 / 单线程下载速度
     例如带宽 10MB/s，单线程 2MB/s，最多 5 个线程
  3. 线程数 ≥ CPU 核心数，≤ CPU 核心数 × 2
     （下载是 IO 密集型，线程可以比 CPU 多）
  4. 内存约束：每线程需要缓冲区
     线程数 × bufferSize ≤ 可用内存 / 安全系数
```

### 2.2 推荐配置

```java
public class DownloadThreadPoolConfig {

    // CPU 核心数
    private static final int CPU_COUNT = Runtime.getRuntime().availableProcessors();

    // 下载场景 —— IO 密集型
    // 核心线程数 = CPU * 2（经验值，可根据实测调整）
    private static final int CORE_POOL_SIZE = Math.max(CPU_COUNT * 2, 4);

    // 最大线程数：给临时突发留余量
    private static final int MAX_POOL_SIZE = CORE_POOL_SIZE * 2;

    // 空闲线程存活时间
    private static final long KEEP_ALIVE_SECONDS = 30L;

    // 阻塞队列 —— 有界，防止无限排队
    // 下载场景任务量已知（等于分块数），用 SynchronousQueue 或小容量队列
    private static final BlockingQueue<Runnable> WORK_QUEUE =
        new LinkedBlockingQueue<>(128);

    /**
     * 创建下载专用线程池
     */
    public static ThreadPoolExecutor createDownloadPool() {
        return new ThreadPoolExecutor(
            CORE_POOL_SIZE,
            MAX_POOL_SIZE,
            KEEP_ALIVE_SECONDS,
            TimeUnit.SECONDS,
            WORK_QUEUE,
            new ThreadFactory() {
                private final AtomicInteger count = new AtomicInteger(1);
                @Override
                public Thread newThread(Runnable r) {
                    Thread t = new Thread(r, "download-" + count.getAndIncrement());
                    t.setDaemon(false);          // 下载不能是守护线程
                    t.setPriority(Thread.NORM_PRIORITY);
                    t.setUncaughtExceptionHandler((thread, e) -> {
                        // 记录失败的分片，后续重试
                        Log.e("Download", "Thread " + thread.getName() + " crash", e);
                    });
                    return t;
                }
            },
            // 拒绝策略：调用者线程执行 —— 下载场景下给一个缓冲机会
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
    }
}
```

### 2.3 为什么这样配置

```
┌─────────────────────┬──────────────────────────────────────────┐
│        参数          │              设置理由                     │
├─────────────────────┼──────────────────────────────────────────┤
│ corePoolSize        │ IO 密集型 = CPU × 2 起步                  │
│    = CPU*2          │ 线程多数时间在等网络 I/O，多开几个不浪费    │
├─────────────────────┼──────────────────────────────────────────┤
│ maxPoolSize         │ 不给太大上限，线程切换本身有开销            │
│    = corePoolSize*2 │ 而且线程数多了可能触发服务端限流            │
├─────────────────────┼──────────────────────────────────────────┤
│ keepAliveTime = 30s │ 空闲线程快速回收，释放资源                 │
├─────────────────────┼──────────────────────────────────────────┤
│ 队列 = LinkedBlocking│ 有界队列防止 OOM，128 足够容纳分片任务      │
│       Queue(128)    │                                          │
├─────────────────────┼──────────────────────────────────────────┤
│ CallerRunsPolicy    │ 队列满时由调用者线程执行，自然的背压机制     │
│                     │ 不会丢任务，也不会无限堆积                  │
├─────────────────────┼──────────────────────────────────────────┤
│ 线程非 daemon        │ 下载任务必须完成，不能被 JVM 提前杀掉       │
└─────────────────────┴──────────────────────────────────────────┘
```

## 三、多线程下载完整实现

### 3.1 分片任务

```java
/**
 * 单个分片下载任务
 */
public class DownloadTask implements Callable<Boolean> {

    private final String url;
    private final long start;     // 起始字节（闭区间）
    private final long end;       // 结束字节（闭区间），-1 表示到文件末尾
    private final File targetFile;
    private final int bufferSize;
    private volatile boolean cancelled = false;

    public DownloadTask(String url, long start, long end,
                        File targetFile, int bufferSize) {
        this.url = url;
        this.start = start;
        this.end = end;
        this.targetFile = targetFile;
        this.bufferSize = bufferSize;
    }

    @Override
    public Boolean call() throws Exception {
        HttpURLConnection conn = null;
        InputStream is = null;
        RandomAccessFile raf = null;

        try {
            conn = (HttpURLConnection) new URL(url).openConnection();
            conn.setConnectTimeout(10_000);
            conn.setReadTimeout(30_000);
            conn.setRequestMethod("GET");

            // 关键：设置 Range 头
            String range = end > 0
                ? "bytes=" + start + "-" + end
                : "bytes=" + start + "-";  // 到尾
            conn.setRequestProperty("Range", range);

            int responseCode = conn.getResponseCode();
            if (responseCode != 206 && responseCode != 200) {
                return false;
            }

            is = new BufferedInputStream(conn.getInputStream(), bufferSize);
            raf = new RandomAccessFile(targetFile, "rw");
            raf.seek(start);

            byte[] buffer = new byte[bufferSize];
            int bytesRead;
            while ((bytesRead = is.read(buffer)) != -1 && !cancelled) {
                raf.write(buffer, 0, bytesRead);
            }
            return !cancelled; // 如果被取消则返回 false
        } finally {
            closeQuietly(is);
            closeQuietly(raf);
            if (conn != null) conn.disconnect();
        }
    }

    public void cancel() {
        cancelled = true;
    }

    private void closeQuietly(Closeable c) {
        if (c != null) {
            try { c.close(); } catch (IOException ignored) { }
        }
    }
}
```

### 3.2 下载管理器（使用线程池）

```java
/**
 * 多线程文件下载器
 */
public class MultiThreadDownloader {

    private final ThreadPoolExecutor pool;
    private final int threadCount;     // 分片数 = 线程数
    private final int bufferSize;      // 每线程缓冲区大小

    // 监听器
    private DownloadListener listener;

    public MultiThreadDownloader() {
        this(DownloadThreadPoolConfig.createDownloadPool(),
             Runtime.getRuntime().availableProcessors() * 2,
             8192); // 8KB buffer
    }

    public MultiThreadDownloader(ThreadPoolExecutor pool,
                                  int threadCount, int bufferSize) {
        this.pool = pool;
        this.threadCount = threadCount;
        this.bufferSize = bufferSize;
    }

    /**
     * 开始下载
     * @param url        文件 URL
     * @param targetFile 目标文件
     * @return 是否成功
     */
    public boolean download(String url, File targetFile) throws Exception {
        // 1. 获取文件大小（HEAD 请求）
        long fileSize = getFileSize(url);
        if (fileSize <= 0) {
            throw new IOException("无法获取文件大小");
        }

        // 2. 预分配文件空间（避免磁盘碎片）
        try (RandomAccessFile raf = new RandomAccessFile(targetFile, "rw")) {
            raf.setLength(fileSize);
        }

        // 3. 计算分片
        long chunkSize = fileSize / threadCount;
        List<DownloadTask> tasks = new ArrayList<>();

        for (int i = 0; i < threadCount; i++) {
            long start = i * chunkSize;
            long end = (i == threadCount - 1) ? fileSize - 1 : start + chunkSize - 1;
            tasks.add(new DownloadTask(url, start, end, targetFile, bufferSize));
        }

        // 4. 提交任务到线程池
        List<Future<Boolean>> futures;
        try {
            futures = pool.invokeAll(tasks);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            // 取消所有任务
            tasks.forEach(DownloadTask::cancel);
            return false;
        }

        // 5. 检查结果
        boolean success = true;
        for (int i = 0; i < futures.size(); i++) {
            try {
                if (!futures.get(i).get()) {
                    System.err.println("分片 " + i + " 下载失败");
                    success = false;
                }
            } catch (ExecutionException e) {
                System.err.println("分片 " + i + " 异常: " + e.getCause().getMessage());
                success = false;
            }
        }

        // 6. 失败则删除文件
        if (!success) {
            targetFile.delete();
        }
        return success;
    }

    /**
     * 获取远程文件大小（HEAD 请求）
     */
    private long getFileSize(String url) throws IOException {
        HttpURLConnection conn = null;
        try {
            conn = (HttpURLConnection) new URL(url).openConnection();
            conn.setRequestMethod("HEAD");
            conn.setConnectTimeout(10_000);
            conn.setReadTimeout(10_000);
            // 部分 CDN 只支持 GET，不支持 HEAD，需要 GET + 只读 Content-Length
            int code = conn.getResponseCode();
            if (code == 200 || code == 206) {
                String header = conn.getHeaderField("Content-Length");
                if (header != null) {
                    return Long.parseLong(header);
                }
            }

            // 兜底：发起 GET 请求，但不读 body
            conn.disconnect();
            conn = (HttpURLConnection) new URL(url).openConnection();
            conn.setRequestMethod("GET");
            conn.setConnectTimeout(10_000);
            conn.setReadTimeout(10_000);
            // 只读头
            conn.getResponseCode();
            String header = conn.getHeaderField("Content-Length");
            if (header != null) {
                return Long.parseLong(header);
            }
            return -1;
        } finally {
            if (conn != null) conn.disconnect();
        }
    }

    public void setListener(DownloadListener listener) {
        this.listener = listener;
    }

    public interface DownloadListener {
        void onProgress(int chunkIndex, long downloaded, long total);
        void onComplete(boolean success);
    }
}
```

## 四、断点续传

```java
/**
 * 支持断点续传的下载器
 * 核心：用临时文件记录每个分片的下载进度
 */
public class ResumableDownloader {

    private final ThreadPoolExecutor pool;
    private final int threadCount;

    public ResumableDownloader(ThreadPoolExecutor pool, int threadCount) {
        this.pool = pool;
        this.threadCount = threadCount;
    }

    /**
     * 下载（自动续传）
     * 进度记录文件：targetFile.progress
     * 格式：每行 "chunkIndex=已下载字节"
     */
    public boolean download(String url, File targetFile) throws Exception {
        File progressFile = new File(targetFile.getParent(),
            targetFile.getName() + ".progress");

        // 读取上次进度
        Map<Integer, Long> completedMap = loadProgress(progressFile);

        // 获取文件大小
        long fileSize = getFileSize(url);
        long chunkSize = fileSize / threadCount;

        // 预分配目标文件（如果不存在）
        if (!targetFile.exists()) {
            try (RandomAccessFile raf = new RandomAccessFile(targetFile, "rw")) {
                raf.setLength(fileSize);
            }
        }

        List<Callable<Boolean>> tasks = new ArrayList<>();
        for (int i = 0; i < threadCount; i++) {
            long chunkStart = i * chunkSize;
            long chunkEnd = (i == threadCount - 1) ? fileSize - 1 : chunkStart + chunkSize - 1;
            long completed = completedMap.getOrDefault(i, 0L);
            long actualStart = chunkStart + completed; // 从断点开始

            if (actualStart > chunkEnd) {
                continue; // 该分片已完成，跳过
            }

            tasks.add(() -> {
                downloadChunk(url, actualStart, chunkEnd, targetFile, i, progressFile, completed);
                return true;
            });
        }

        if (tasks.isEmpty()) {
            progressFile.delete(); // 所有分片已完成
            return true;
        }

        List<Future<Boolean>> futures = pool.invokeAll(tasks);
        // 检查结果...
        return true;
    }

    private void downloadChunk(String url, long start, long end,
                                File targetFile, int chunkIndex,
                                File progressFile, long alreadyCompleted)
            throws IOException {
        HttpURLConnection conn = (HttpURLConnection) new URL(url).openConnection();
        conn.setRequestProperty("Range", "bytes=" + start + "-" + end);

        try (InputStream is = conn.getInputStream();
             RandomAccessFile raf = new RandomAccessFile(targetFile, "rw")) {

            raf.seek(start);
            byte[] buffer = new byte[8192];
            long downloaded = alreadyCompleted;
            int bytesRead;

            while ((bytesRead = is.read(buffer)) != -1) {
                raf.write(buffer, 0, bytesRead);
                downloaded += bytesRead;

                // 每 100KB 更新一次进度文件
                if (downloaded % (100 * 1024) == 0) {
                    updateProgress(progressFile, chunkIndex, downloaded);
                }
            }
            // 分片完成，从进度文件移除该条目
            removeProgress(progressFile, chunkIndex);
        } finally {
            conn.disconnect();
        }
    }

    // ---- 进度文件操作 ----

    private Map<Integer, Long> loadProgress(File progressFile) {
        Map<Integer, Long> map = new HashMap<>();
        if (!progressFile.exists()) return map;

        try (BufferedReader reader = new BufferedReader(new FileReader(progressFile))) {
            String line;
            while ((line = reader.readLine()) != null) {
                String[] parts = line.split("=");
                if (parts.length == 2) {
                    map.put(Integer.parseInt(parts[0]), Long.parseLong(parts[1]));
                }
            }
        } catch (Exception e) {
            // 进度文件损坏则放弃进度
        }
        return map;
    }

    private synchronized void updateProgress(File progressFile,
                                              int chunkIndex, long downloaded) {
        // 将 Map 序列化回文件
    }

    private synchronized void removeProgress(File progressFile, int chunkIndex) {
        // 从文件中移除该行
    }
}
```

## 六、常见坑 & 注意事项

```
  ┌─────────────────────────────────────────────────────────────┐
  │  1. 服务端不支持 Range 请求                                   │
  │     → 先发 HEAD 请求检查 Accept-Ranges: bytes                │
  │     → 不支持则降级为单线程下载                                │
  ├─────────────────────────────────────────────────────────────┤
  │  2. 文件合并时顺序不对                                        │
  │     → 使用 RandomAccessFile.seek() 直接写对应偏移             │
  │     → 不需要最后再合并，每个线程各写各的位置                    │
  ├─────────────────────────────────────────────────────────────┤
  │  3. 线程池 shutdown 后还有任务在执行                           │
  │     → invokeAll 会等待所有任务完成，或超时后取消               │
  │     → 下载完成后再 shutdown()，或在 finally 中 shutdownNow()   │
  ├─────────────────────────────────────────────────────────────┤
  │  4. 线程数太多被服务端限流                                     │
  │     → 控制总线程数 ≤ 8-16                                     │
  │     → 避免多任务同时下载时线程池过大                           │
  ├─────────────────────────────────────────────────────────────┤
  │  5. 文件已存在但大小不对                                       │
  │     → 检查本地文件大小，若与服务端一致则跳过                   │
  │     → 若不一致则删除重下或从断点续传                          │
  ├─────────────────────────────────────────────────────────────┤
  │  6. 网络异常导致部分分片失败                                   │
  │     → 失败的分片单独重试（只重试失败的，不是整个文件重下）      │
  │     → 重试3次还不行则标记下载失败                             │
  └─────────────────────────────────────────────────────────────┘
```

## 七、面试题

**Q1: 多线程下载大文件，线程数怎么确定？**

> 主观参数，但有约束：① 线程数 ≤ 服务端单 IP 最大连接数（防止被 ban）② IO 密集型，线程数可以 = CPU 核心数 × 2~4，因为大部分时间在等网络 ③ 线程数 × 缓冲区大小 ≤ 可用内存 ④ 分片数 = 线程数，分片太小反而 header 开销占比大。一般设 4~8 个线程即可，再多边际收益递减。

**Q2: `invokeAll` vs 逐个 `submit` 有什么不同？**

> `invokeAll` 是阻塞的，等所有 Callable 都返回才继续，适合下载这种"所有分片都必须完成"的场景。逐个 `submit` 返回 `Future`，可通过 `Future.get()` 逐个等待，好处是可以逐个拿到结果、失败时及时取消其他任务，更灵活但代码更复杂。

**Q3: IO 密集型和 CPU 密集型任务，线程池配置有什么区别？**

> **CPU 密集型**：线程数 = CPU 核心数 + 1，多一个用于补偿因缺页等因素导致的暂停。多了反而上下文切换开销大。
> **IO 密集型**：线程数 = CPU 核心数 × 2~N，N 取决于等待时间占比。如果等 IO 的时间占 90%，理论上可以开到 CPU × 10，但实际受内存和连接数限制，一般取 × 2~4。

**Q4: 为什么不直接用 CachedThreadPool 做下载？**

> CachedThreadPool 遇一个任务开一个线程，下载大文件时如果同时切 100 个分片，会直接开 100 个线程。线程过多导致：① 每个线程分到的带宽太小 ② 上下文切换开销反而降低吞吐 ③ 可能触发服务端限流。下载场景线程数应该可控且有限。
