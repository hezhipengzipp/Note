# Flutter 一对多事件分发 —— 登录状态同步

## 一、问题场景

```
  登录成功后，哪些地方需要感知？

        ┌────────────────┐
        │   登录成功后     │
        └───────┬────────┘
                │
   ┌────────────┼────────────┐
   │            │            │
   ▼            ▼            ▼
 首页刷新    抽屉头更新    个人中心
 "立即登录"   用户头像     刷新数据
 变为内容页   变为登录态    请求用户信息
                │
                ▼
            其他页面需要
            退出的都要感知
```

**核心需求：A 事件发生 → 多个 B/C/D 自动响应，且 B/C/D 之间互不耦合。**

## 三、方案一：EventBus（最轻量）

### 3.1 核心实现

```dart
/// 事件总线 —— 基于 StreamController
class EventBus {
  // Broadcast 允许多个监听者
  final _controller = StreamController<dynamic>.broadcast();

  /// 发送事件
  void fire(dynamic event) => _controller.add(event);

  /// 监听某类事件（泛型筛选）
  Stream<T> on<T>() => _controller.stream.where((e) => e is T).cast<T>();

  /// 监听所有事件
  Stream<dynamic> get stream => _controller.stream;

  void dispose() => _controller.close();
}
```

### 3.2 使用方式

```dart
// ===== 定义事件 =====
class LoginSuccessEvent {
  final String userId;
  final String token;
  LoginSuccessEvent(this.userId, this.token);
}

class LogoutEvent {}

// ===== 登录页 —— 发送事件 =====
class LoginPage extends StatelessWidget {
  final EventBus eventBus; // 通过 DI 或构造函数注入

  Future<void> login() async {
    final result = await AuthService.login(phone, code);
    // 登录成功，通知其他页面
    eventBus.fire(LoginSuccessEvent(result.userId, result.token));
  }

  Widget build(BuildContext context) { /* ... */ }
}

// ===== 首页 —— 监听事件 =====
class HomePage extends StatefulWidget {
  @override
  State<HomePage> createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  StreamSubscription? _sub;

  @override
  void initState() {
    super.initState();
    final bus = GetIt.I<EventBus>(); // 从 DI 获取全局单例
    _sub = bus.on<LoginSuccessEvent>().listen((event) {
      // 收到登录事件，刷新首页数据
      loadUserData();
    });
    // 也监听登出
    bus.on<LogoutEvent>().listen((_) {
      clearUserData();
    });
  }

  @override
  void dispose() {
    _sub?.cancel();  // 重要：页面销毁时取消订阅
    super.dispose();
  }
}
```

### 3.3 优缺点

```
  ✅ 极简实现，10 行代码
  ✅ 发送方和接收方完全解耦
  ✅ 事件类型通过泛型天然隔离

  ❌ 没有"当前状态"的概念（只能感知变化，不能直接读值）
  ❌ 需要手动管理 StreamSubscription（忘记 dispose 会内存泄漏）
  ❌ 事件多了后类型管理混乱
```

## 四、方案二：全局 ChangeNotifier + Provider

```dart
/// 登录状态管理 —— 同时具备"事件通知"和"状态读取"能力
class AuthState extends ChangeNotifier {
  String? _userId;
  String? _token;
  bool _isLoggedIn = false;

  bool get isLoggedIn => _isLoggedIn;
  String? get userId => _userId;

  /// 登录
  Future<void> login(String phone, String code) async {
    final result = await AuthService.login(phone, code);
    _userId = result.userId;
    _token = result.token;
    _isLoggedIn = true;
    notifyListeners();  // ← 通知所有监听者
  }

  /// 登出
  Future<void> logout() async {
    _userId = null;
    _token = null;
    _isLoggedIn = false;
    notifyListeners();
  }
}

// ===== Provider 注入（App 顶层） =====
void main() {
  runApp(
    MultiProvider(
      providers: [
        ChangeNotifierProvider(create: (_) => AuthState()),
      ],
      child: MyApp(),
    ),
  );
}

// ===== 首页 —— 自动响应 =====
class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    // 写法1：watch 监听整个 AuthState
    final auth = context.watch<AuthState>();

    // 写法2：只监听 isLoggedIn
    final isLoggedIn = context.select<AuthState, bool>((a) => a.isLoggedIn);

    return isLoggedIn ? ContentView() : LoginPrompt();
  }
}

// ===== 登录页 —— 调用 login =====
class LoginPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final auth = context.read<AuthState>(); // read 不监听，只获取引用
    return ElevatedButton(
      onPressed: () => auth.login(phone, code),
      child: Text('登录'),
    );
  }
}
```

## 五、方案三：BLoC 全局 Stream

```dart
// ===== 全局 AuthCubit =====
class AuthCubit extends Cubit<AuthState> {
  AuthCubit() : super(AuthState.unknown());

  Future<void> login(String phone, String code) async {
    emit(AuthState.loading());
    try {
      final result = await AuthService.login(phone, code);
      emit(AuthState.authenticated(result.userId, result.token));
    } catch (e) {
      emit(AuthState.error(e.toString()));
    }
  }

  Future<void> logout() async {
    emit(AuthState.unknown());
  }
}

// ===== 登录页：发送 =====
class LoginPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return BlocListener<AuthCubit, AuthState>(
      listener: (context, state) {
        if (state is Authenticated) {
          // 登录成功，触发路由跳转（一次性副作用）
          Navigator.of(context).pushReplacementNamed('/home');
        }
      },
      child: /* UI */,
    );
  }
}

// ===== 首页：接收 =====
class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return BlocBuilder<AuthCubit, AuthState>(
      builder: (context, state) {
        return state.when(
          authenticated: (userId, token) => ContentView(userId: userId),
          unknown: () => LoginPrompt(),
          loading: () => CircularProgressIndicator(),
          error: (msg) => ErrorView(msg),
        );
      },
    );
  }
}

// ===== 抽屉头部：接收 =====
class DrawerHeader extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    // 只监听 userId
    final userId = context.select<AuthCubit, String?>(
      (cubit) => cubit.state is Authenticated
          ? (cubit.state as Authenticated).userId
          : null,
    );
    return userId != null
        ? UserAvatar(userId: userId)
        : Text('请登录');
  }
}
```

## 六、方案四：GetX 响应式

```dart
// ===== 全局 AuthController =====
class AuthController extends GetxController {
  final isLoggedIn = false.obs;
  final userId = Rxn<String>();     // 可空 Rx
  final token = Rxn<String>();

  Future<void> login(String phone, String code) async {
    final result = await AuthService.login(phone, code);
    userId.value = result.userId;
    token.value = result.token;
    isLoggedIn.value = true;
  }

  Future<void> logout() async {
    userId.value = null;
    token.value = null;
    isLoggedIn.value = false;
  }
}

// ===== 登录页：发送 =====
class LoginPage extends GetView<AuthController> {
  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: () => controller.login(phone, code),
      child: Text('登录'),
    );
  }
}

// ===== 首页：接收 =====
class HomePage extends StatelessWidget {
  final controller = Get.find<AuthController>();

  @override
  Widget build(BuildContext context) {
    return Obx(() {
      return controller.isLoggedIn.value
          ? ContentView()
          : LoginPrompt();
    });
  }
}

// ===== 抽屉头部：接收 =====
class DrawerHeader extends StatelessWidget {
  final controller = Get.find<AuthController>();

  @override
  Widget build(BuildContext context) {
    return Obx(() {
      final id = controller.userId.value;
      return id != null
          ? UserAvatar(userId: id)
          : Text('请登录');
    });
  }
}
```

## 七、方案五：纯 StreamController（无框架依赖）

适合不想引入 BLoC/Provider/GetX 的项目：

```dart
/// 全局单例 —— 登录状态流
class LoginStateNotifier {
  LoginStateNotifier._();
  static final instance = LoginStateNotifier._();

  final _controller = StreamController<LoginEvent>.broadcast();

  Stream<LoginEvent> get event => _controller.stream;

  void notify(LoginEvent event) => _controller.add(event);

  void dispose() => _controller.close();
}

sealed class LoginEvent {}
class LoggedIn extends LoginEvent {
  final String userId;
  LoggedIn(this.userId);
}
class LoggedOut extends LoginEvent {}

// ===== 发送 =====
LoginStateNotifier.instance.notify(LoggedIn('user123'));

// ===== 接收 =====
Subscription<LoginEvent>? _sub;

@override
void initState() {
  super.initState();
  _sub = LoginStateNotifier.instance.event.listen((event) {
    switch (event) {
      case LoggedIn(userId: final id):
        loadUserData(id);
      case LoggedOut():
        clearUserData();
    }
  });
}

@override
void dispose() {
  _sub?.cancel();
  super.dispose();
}
```

## 九、推荐选择

```
  我是这样选的，按优先级：

  项目已在用 BLoC
      → 直接全局 AuthCubit，BlocBuilder 监听，最简单

  项目已在用 Provider
      → ChangeNotifier，Context.watch / context.select

  项目已在用 GetX
      → .obs + Obx

  项目没用任何状态管理 / 想要最小侵入
      → EventBus（如果只需要事件通知）
      → 纯 StreamController（如果接受手动管理生命周期）

  新项目
      → BLoC（事件驱动天然适合这种场景）
```

核心原则就一条：**让"需要知道登录状态"的地方主动订阅，而不是"触发登录的地方"去逐个通知。**

## 十、面试题

**Q1: Flutter 中如何实现跨页面的事件通知？有哪些方案？**

> ① **EventBus**：基于 `StreamController.broadcast()`，发布/订阅模式，解耦彻底
> ② **Provider / ChangeNotifier**：全局 `ChangeNotifier` + `context.watch`，既能读当前状态又能感知变化
> ③ **BLoC**：全局 `Cubit`/`Bloc` + `BlocBuilder`/`BlocListener`，事件驱动
> ④ **纯 Stream**：自己维护一个 `StreamController.broadcast()`
>
> 本质都是一样的——观察者模式。区别在于是否自带"当前状态"、是否自动管理 Subscription 生命周期、框架依赖程度。

**Q2: 为什么说 StreamSubscription 需要 dispose？忘记 dispose 会怎样？**

> `StreamController.broadcast()` 内部维护了一个监听者列表。每个 `listen()` 返回的 `StreamSubscription` 持有指向这个列表的引用。如果 Widget dispose 了但 Subscription 还在，会：
> ① 内存泄漏（Subscription + Widget State 都无法回收）
> ② 可能对已不可见的 Widget 调用 `setState()`，报错或行为错乱
>
> Provider 和 BLoC 内部已通过 `BuildContext` 生命周期自动管理，所以不需要手动 dispose。

**Q3: EventBus 用 `dynamic` 做事件，类型不安全怎么办？**

> 两种做法：
> ① 用 `StreamController<T>()` 的 `.where((e) => e is T).cast<T>()` 泛型过滤，如 `bus.on<LoginEvent>()`
> ② 定义 sealed class 做事件家族：`sealed class AppEvent {}`，`class LoginEvent extends AppEvent {}`，避免用 `dynamic` 满天飞
