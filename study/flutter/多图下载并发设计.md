# Flutter 多图下载并发设计

---

## 一、方案一：Future.wait 全部并发（最简单）

```dart
final urls = ['url1', 'url2', 'url3', 'url4', 'url5'];

// 5 个请求同时发出，全部完成后返回
final results = await Future.wait(
  urls.map((url) => downloadImage(url)),
);
```

```
时间线：

  Future.wait 同时发起所有请求：

  url1  ├──── 下载中 ────┤ 完成
  url2  ├─── 下载中 ───┤ 完成
  url3  ├────── 下载中 ──────┤ 完成
  url4  ├── 下载中 ──┤ 完成
  url5  ├───── 下载中 ─────┤ 完成
                                 ↑
                          全部完成才返回

  问题：
  100 张图同时发 → 100 个 TCP 连接 → 内存暴涨 + 服务器压力大
```

**适合**：数量少（< 10 张）的场景。

---

## 二、方案二：控制并发数（实际项目用这个）

同时最多 N 个下载，完成一个再启动下一个：

```dart
/// 并发控制器：限制同时执行的任务数
Future<List<T>> parallelLimit<T>(
  List<Future<T> Function()> tasks,
  int maxConcurrent,
) async {
  final results = <T?>List.filled(tasks.length, null);
  var nextIndex = 0;

  Future<void> worker() async {
    while (nextIndex < tasks.length) {
      final i = nextIndex++;
      results[i] = await tasks[i]();
    }
  }

  // 启动 maxConcurrent 个 worker 并发执行
  await Future.wait(
    List.generate(maxConcurrent, (_) => worker()),
  );

  return results.cast<T>();
}

// 使用：最多 3 个并发下载
final images = await parallelLimit(
  urls.map((url) => () => downloadImage(url)).toList(),
  3,  // 最大并发数
);
```

```
时间线（并发数 = 3）：

  槽位1:  url1 ├── 下载 ──┤  url4 ├── 下载 ──┤
  槽位2:  url2 ├─ 下载 ─┤  url5 ├── 下载 ────┤
  槽位3:  url3 ├─── 下载 ───┤  url6 ├─ 下载 ─┤

  同时最多 3 个在跑
  一个完成 → 下一个立刻顶上
  不会同时开 100 个连接
```

## 三、方案三：Stream 实现（带进度回调）

```dart
Stream<(int index, Uint8List data)> downloadAll(
  List<String> urls, {
  int maxConcurrent = 3,
}) async* {
  final pool = <Future<(int, Uint8List)>>[];
  var nextIndex = 0;

  // 填满并发池
  while (nextIndex < urls.length || pool.isNotEmpty) {
    // 补充任务到并发池
    while (pool.length < maxConcurrent && nextIndex < urls.length) {
      final i = nextIndex++;
      pool.add(
        downloadImage(urls[i]).then((data) => (i, data)),
      );
    }

    // 等任意一个完成
    final completed = await Future.any(pool);
    pool.removeWhere((f) => f == completed); // 简化示意
    yield completed;  // 每完成一个就通知外部
  }
}

// 使用：实时获取每张图的下载结果
await for (var (index, data) in downloadAll(urls)) {
  print('第 $index 张下载完成，${data.length} bytes');
  // 可以立刻显示这张图，不用等全部完成
}
```

## 四、方案四：完整的生产级实现

```dart
class ImageDownloader {
  final int maxConcurrent;
  int _running = 0;
  final _queue = <_Task>[];

  ImageDownloader({this.maxConcurrent = 3});

  Future<Uint8List> download(String url) {
    final completer = Completer<Uint8List>();
    _queue.add(_Task(url, completer));
    _tryRunNext();
    return completer.future;
  }

  void _tryRunNext() {
    while (_running < maxConcurrent && _queue.isNotEmpty) {
      _running++;
      final task = _queue.removeAt(0);
      _execute(task);
    }
  }

  Future<void> _execute(_Task task) async {
    try {
      final data = await _doDownload(task.url);
      task.completer.complete(data);
    } catch (e) {
      task.completer.completeError(e);
    } finally {
      _running--;
      _tryRunNext();  // 完成一个，启动下一个
    }
  }

  Future<Uint8List> _doDownload(String url) async {
    final response = await http.get(Uri.parse(url));
    if (response.statusCode != 200) {
      throw Exception('下载失败: ${response.statusCode}');
    }
    return response.bodyBytes;
  }
}

class _Task {
  final String url;
  final Completer<Uint8List> completer;
  _Task(this.url, this.completer);
}
```

```dart
// 使用
final downloader = ImageDownloader(maxConcurrent: 3);

// 提交 100 个下载任务，同时最多 3 个在跑
final futures = urls.map((url) => downloader.download(url)).toList();

// 带进度
for (var i = 0; i < futures.length; i++) {
  futures[i].then((data) {
    setState(() {
      images[i] = data;
      progress = completedCount / total;
    });
  });
}

// 或者等全部完成
final allImages = await Future.wait(futures);
```

### 内部工作流程

```
  提交 url1 → _running=1，立即执行
  提交 url2 → _running=2，立即执行
  提交 url3 → _running=3，立即执行
  提交 url4 → _running=3 已满，进队列等待
  提交 url5 → 进队列等待

  url2 完成 → _running=2 → 从队列取 url4 执行 → _running=3
  url1 完成 → _running=2 → 从队列取 url5 执行 → _running=3
  ...
```

## 五、和底层的关系

```
所有方案底层都是一样的：

  Dart 层（单线程）
  ┌──────────────────────────────────────────┐
  │                                          │
  │  发起 3 个 http.get                       │
  │  → 3 个 Future 注册到 Event Loop          │
  │  → 函数 await 让出线程                    │
  │                                          │
  │  Event Loop 继续处理 UI 刷新、动画         │
  │                                          │
  └──────────────────────────────────────────┘
          │  │  │
          ▼  ▼  ▼  （3 个 socket fd 注册到 epoll）

  IO 线程（C++ 层）
  ┌──────────────────────────────────────────┐
  │  epoll_wait(fd1, fd2, fd3)               │
  │  同时监听 3 个 socket                     │
  │                                          │
  │  fd2 数据到了 → PostTask 回 UI Runner     │
  │  fd1 数据到了 → PostTask 回 UI Runner     │
  │  fd3 数据到了 → PostTask 回 UI Runner     │
  └──────────────────────────────────────────┘

  "并发控制"只是控制同时注册多少个 fd 到 epoll
  真正的网络 IO 都在 C++ IO 线程并行处理
  Dart 单线程不受影响
```

## 六、方案选型

| 方案 | 适用场景 | 复杂度 |
|------|---------|-------|
| `Future.wait` 全并发 | < 10 个任务 | 最低 |
| `parallelLimit` 函数 | 简单场景控制并发 | 低 |
| `Stream` + 并发池 | 需要逐个回调进度 | 中 |
| `ImageDownloader` 类 | 生产项目，需要队列管理 | 中 |
| `Dio` + 拦截器 | 已用 Dio 的项目 | 低（Dio 内置） |

## 七、面试回答模板

**Q: Flutter 中怎么设计多图并发下载？**

> 不能用 Future.wait 无限制并发，100 张图同时开 100 个连接会内存暴涨。正确做法是控制并发数，比如同时最多 3~5 个下载。实现上用一个任务队列和计数器：提交任务时如果正在运行的数量未达上限就立即执行，达到上限就进队列等待；每个任务完成后计数器减一，从队列取下一个任务执行。底层原理是控制同时注册到 epoll 的 socket fd 数量，真正的网络 IO 在 C++ IO 线程并行处理，Dart 单线程只负责调度，不会阻塞 UI。这个模式和 Android OkHttp 的 Dispatcher 限制最大并发请求数是同一个思路。
