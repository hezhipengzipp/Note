# Flutter 并发请求控制：固定并发数

> 场景：1000 个请求，始终保持 5 个并发在执行，一个回来立即发起下一个。

---

## 一、核心思路：Worker 模式

```
1000 个任务排队，启动 5 个 Worker 同时去取任务执行：

  任务队列: [task0, task1, task2, ..., task999]
              ↑      ↑      ↑      ↑      ↑
            Worker0 Worker1 Worker2 Worker3 Worker4

  每个 Worker：取一个 → 执行 → 完成 → 再取一个 → 执行 → ...直到队列空

  Worker 之间互不影响，每个 Worker 串行执行自己取到的任务
  5 个 Worker 并发执行 → 同时最多 5 个请求在线
```

---

## 二、实现一：Worker 取任务（最简洁推荐）

```dart
/// 并发控制器
/// [tasks] 任务工厂列表（每个元素是一个返回 Future 的函数）
/// [maxConcurrent] 最大并发数
Future<List<T>> runWithConcurrency<T>(
  List<Future<T> Function()> tasks,
  int maxConcurrent,
) async {
  final results = <T?>List.filled(tasks.length, null);
  var nextIndex = 0;

  Future<void> worker() async {
    while (nextIndex < tasks.length) {
      final index = nextIndex++;    // 原子地取下一个任务下标
      results[index] = await tasks[index]();
    }
  }

  // 启动 maxConcurrent 个 worker，一起抢活干
  await Future.wait(
    List.generate(maxConcurrent, (_) => worker()),
  );

  return results.cast<T>();
}
```

```dart
// 使用：模拟 1000 个请求，最多 5 个并发
Future<void> main() async {
  final tasks = List.generate(1000, (i) => () async {
    print('请求 $i 开始');
    await Future.delayed(Duration(milliseconds: 100)); // 模拟网络请求
    print('请求 $i 完成');
    return '结果 $i';
  });

  final results = await runWithConcurrency(tasks, 5);
  print('全部完成，共 ${results.length} 条结果');
}
```

```
时间线（1000 个任务，5 并发）：

  Worker0: task0├─执行─┤ task5├─执行─┤ task10├─执行─┤ ...
  Worker1: task1├─执行─┤ task6├─执行─┤ task11├─执行─┤ ...
  Worker2: task2├─执行─┤ task7├─执行─┤ task12├─执行─┤ ...
  Worker3: task3├─执行─┤ task8├─执行─┤ task13├─执行─┤ ...
  Worker4: task4├─执行─┤ task9├─执行─┤ task14├─执行─┤ ...

  任何时候最多 5 个在执行
  任意一个完成 → 立刻取下一个任务补上
```

### 要点

- `nextIndex++` 是单线程递增，没有竞态问题（Dart 单线程）
- 不需要锁、不需要 Completer，不到 20 行代码
- `tasks[i]()` 是工厂函数，调用时才发起请求，不是提前发起

## 三、实现二：队列 + 计数器（更灵活）

适合需要动态添加任务的场景：

```dart
class ConcurrencyPool {
  final int maxConcurrent;
  int _running = 0;
  final _queue = <Future<void> Function()>[];

  ConcurrencyPool(this.maxConcurrent);

  /// 添加一个任务，返回该任务的 Future
  Future<T> add<T>(Future<T> Function() task) {
    final completer = Completer<T>();
    _queue.add(() async {
      try {
        completer.complete(await task());
      } catch (e) {
        completer.completeError(e);
      } finally {
        _running--;
        _tryRunNext();
      }
    });
    _tryRunNext();
    return completer.future;
  }

  void _tryRunNext() {
    while (_running < maxConcurrent && _queue.isNotEmpty) {
      _running++;
      _queue.removeAt(0)();
    }
  }
}
```

```dart
// 使用
final pool = ConcurrencyPool(5);

// 逐个添加 1000 个任务
final futures = List.generate(1000, (i) {
  return pool.add(() async {
    await Future.delayed(Duration(milliseconds: 100));
    return '结果 $i';
  });
});

// 等全部完成
final results = await Future.wait(futures);
```

```
内部状态变化（maxConcurrent = 5）：

  添加 task0 → _running:1  执行
  添加 task1 → _running:2  执行
  添加 task2 → _running:3  执行
  添加 task3 → _running:4  执行
  添加 task4 → _running:5  执行
  添加 task5 → _running:5  已满 → 入队
  添加 task6 → _running:5  已满 → 入队
  ...
  task2 完成 → _running:4 → 取 task5 执行 → _running:5
  task0 完成 → _running:4 → 取 task6 执行 → _running:5
  ...
```

**和方案一区别**：这个可以随时动态添加任务，方案一必须一开始就确定任务列表。

## 四、实现三：Stream 逐个回调（带进度）

需要在每个任务完成时立刻通知 UI：

```dart
Stream<T> runWithConcurrencyStream<T>(
  List<Future<T> Function()> tasks,
  int maxConcurrent,
) async* {
  var nextIndex = 0;
  final completers = <int, Completer<void>>{};

  Future<void> worker(int workerId) async {
    while (nextIndex < tasks.length) {
      final index = nextIndex++;
      final completer = Completer<void>();
      completers[index] = completer;
      final result = await tasks[index]();
      yield result;         // 每完成一个就通知外部
      completer.complete();
    }
  }

  // 注意：async* 内不能直接用 Future.wait + yield，
  // 实际生产建议用 StreamController 实现，见下方。
}
```

上方的 async* 版有语法限制，实际工程用 `StreamController`：

```dart
Stream<T> runWithConcurrencyStream<T>(
  List<Future<T> Function()> tasks,
  int maxConcurrent,
) {
  final controller = StreamController<T>();
  var nextIndex = 0;
  var completedCount = 0;

  void worker() async {
    while (nextIndex < tasks.length) {
      final index = nextIndex++;
      try {
        final result = await tasks[index]();
        controller.add(result);
      } catch (e) {
        controller.addError(e);
      }
      completedCount++;
    }
  }

  // 启动 workers
  for (var i = 0; i < maxConcurrent; i++) {
    worker();
  }

  // 监听完成（简化：实际需精确判断全部完成）
  return controller.stream;
}
```

```dart
// 使用：完成一个更新一次 UI
final stream = runWithConcurrencyStream(tasks, 5);
var doneCount = 0;

stream.listen(
  (result) {
    doneCount++;
    print('进度: $doneCount / 1000, 结果: $result');
  },
  onDone: () => print('全部完成'),
);
```

## 五、进阶：支持取消和重试

```dart
class AdvancedConcurrencyPool {
  final int maxConcurrent;
  int _running = 0;
  final _queue = <_Task>[];
  bool _cancelled = false;

  AdvancedConcurrencyPool(this.maxConcurrent);

  Future<T> add<T>({
    required Future<T> Function() task,
    int maxRetries = 0,
  }) {
    final completer = Completer<T>();
    _queue.add(_Task(
      () async {
        var retries = 0;
        while (true) {
          try {
            completer.complete(await task());
            return;
          } catch (e) {
            if (retries >= maxRetries || _cancelled) {
              completer.completeError(e);
              return;
            }
            retries++;
          }
        }
      },
      completer,
    ));
    _tryRunNext();
    return completer.future;
  }

  void cancelAll() {
    _cancelled = true;
    for (var task in _queue) {
      task.completer.completeError(CancelledException());
    }
    _queue.clear();
  }

  void _tryRunNext() {
    while (_running < maxConcurrent && _queue.isNotEmpty && !_cancelled) {
      _running++;
      final task = _queue.removeAt(0);
      task.fn().whenComplete(() {
        _running--;
        _tryRunNext();
      });
    }
  }
}

class _Task<T> {
  final Future<void> Function() fn;
  final Completer<T> completer;
  _Task(this.fn, this.completer);
}

class CancelledException implements Exception {}
```

## 六、和 Dart 异步模型的关系

```
Worker 模式之所以能工作，是因为 Dart 单线程 Event Loop：

  nextIndex++ 时：
  ┌─────────────────────────────────────────────────────┐
  │  所有 Worker 跑在同一个 Isolate 的同一个线程上       │
  │                                                     │
  │  Worker0: nextIndex++ → 拿到 0 → await → 让出线程   │
  │  Worker1: nextIndex++ → 拿到 1 → await → 让出线程   │
  │  Worker2: nextIndex++ → 拿到 2 → await → 让出线程   │
  │  ...                                                │
  │                                                     │
  │  await 期间 Event Loop 去处理其他事件                │
  │  回调回来时 Event Loop 调度对应 Future 继续执行      │
  │                                                     │
  │  每个 await 点都是一次"主动让出"                     │
  │  nextIndex++ 是同步操作，不可能被打断                │
  └─────────────────────────────────────────────────────┘

  5 个 worker 并不是 5 个线程
  而是 5 个"协程"在同一个线程上交替执行
  真正的 IO 在 C++ 层的 IO Runner 上并行
```

## 七、方案对比

| 方案 | 代码量 | 适用场景 | 动态添加 | 取消 |
|------|-------|---------|---------|------|
| Worker 模式 | ~15 行 | 任务列表固定 | 否 | 否 |
| 队列+计数器 | ~30 行 | 需要动态加任务 | 是 | 否 |
| Stream 版 | ~35 行 | 需要逐个回调进度 | 否 | 否 |
| 完整版 | ~60 行 | 生产项目 | 是 | 是 |

## 八、面试回答模板

**Q: 1000 个请求，要求同时最多 5 个并发，怎么实现？**

> 核心是 Worker 模式：用一个原子递增的索引 `nextIndex` 和一个任务列表，启动 5 个 async 函数作为 Worker，每个 Worker 在 while 循环里取任务——用 `nextIndex++` 原子地拿到自己该执行的任务下标，然后 await 执行，完成后循环取下一个，直到队列空。因为是 Dart 单线程，`nextIndex++` 不会被其他 Worker 打断，天然线程安全。本质上是把 1000 个任务的"调度"控制在 Dart 层，真正并发的是底层 IO。
>
> 如果需要动态添加任务，用队列+计数器模式：维护一个任务队列和一个 `_running` 计数器，每次添加任务时判断 `_running < maxConcurrent`，是则立即执行，否则入队；每个任务完成时 `_running--` 并从队列取下一个。
