# Flutter 屏幕适配方案

---

## 一、核心原理

```
比例缩放公式：实际值 = 设计稿值 × 屏幕宽度 / 设计稿宽度

所有适配方案本质都是这个公式，只是封装程度不同。
```

---

## 二、方案 1：MediaQuery（基础）

```dart
final size = MediaQuery.of(context).size;
final width = size.width;    // 屏幕宽度（逻辑像素）
final height = size.height;  // 屏幕高度
final dpr = MediaQuery.of(context).devicePixelRatio;  // 像素比
final statusBar = MediaQuery.of(context).padding.top;  // 状态栏高度
```

---

## 三、方案 2：自定义 Adapter（等比缩放）

```dart
class ScreenAdapter {
  static late double _designWidth;
  static late MediaQueryData _mediaQuery;

  static void init(BuildContext context, {double designWidth = 375}) {
    _designWidth = designWidth;
    _mediaQuery = MediaQuery.of(context);
  }

  static double w(double width) {
    return width * _mediaQuery.size.width / _designWidth;
  }

  static double h(double height) {
    return height * _mediaQuery.size.height / _designWidth;
  }

  static double sp(double fontSize) {
    return fontSize * _mediaQuery.size.width / _designWidth;
  }
}

// 使用
ScreenAdapter.init(context, designWidth: 375);
Container(
  width: ScreenAdapter.w(100),
  height: ScreenAdapter.h(50),
  child: Text('Hello', style: TextStyle(fontSize: ScreenAdapter.sp(16))),
)
```

## 四、方案 3：flutter_screenutil（推荐库）

```yaml
dependencies:
  flutter_screenutil: ^5.9.0
```

```dart
// 初始化
class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ScreenUtilInit(
      designSize: Size(375, 812),  // 设计稿尺寸
      builder: (context, child) {
        return MaterialApp(home: HomePage());
      },
    );
  }
}

// 使用
Container(
  width: 100.w,           // 设计稿 100px
  height: 50.h,           // 设计稿 50px
  margin: EdgeInsets.all(10.r),  // 圆角/间距
  child: Text(
    'Hello',
    style: TextStyle(fontSize: 16.sp),  // 字体大小
  ),
)
```

### 自定义 Adapter vs flutter_screenutil

```
原理完全一样：设计稿值 × 屏幕宽度 / 设计稿宽度

区别在于封装程度：
```

| | 自定义 Adapter | flutter_screenutil |
|---|---|---|
| 核心原理 | 比例缩放 | 比例缩放 |
| 初始化 | 手动 init(context) | ScreenUtilInit 自动 |
| 扩展方法 | 自己写 .w .h .sp | 内置 .w .h .sp .r |
| 横竖屏 | 需要自己处理 | 自动监听 |
| 最大最小值 | 自己限制 | 内置 minWidth/maxWidth |
| 桌面适配 | 自己处理 | scaleByHeight 选项 |

## 六、方案 5：LayoutBuilder（响应式布局）

```dart
LayoutBuilder(
  builder: (context, constraints) {
    if (constraints.maxWidth > 600) {
      return _buildWideLayout();    // 平板/桌面
    } else {
      return _buildNarrowLayout();  // 手机
    }
  },
)
```

## 七、方案 6：百分比 / 固定比例

```dart
// 百分比宽度
FractionallySizedBox(
  widthFactor: 0.8,   // 占父容器 80%
```

## 八、安全区域适配

```dart
  resizeToAvoidBottomInset: true,  // 默认 true
```

## 九、各方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|---|---|---|---|
| MediaQuery | 获取屏幕信息 | 原生 API | 需要手动计算 |
| 自定义 Adapter | 等比缩放 | 灵活可控 | 需要封装 |
| flutter_screenutil | 等比缩放 | 开箱即用 | 依赖库 |
| Expanded/Flexible | 比例布局 | 原生支持 | 只能比例 |
| LayoutBuilder | 响应式 | 按宽度适配 | 逻辑复杂 |
| FractionallySizedBox | 百分比 | 原生支持 | 只能百分比 |

## 十一、面试常问

### 1. flutter_screenutil 和自定义 Adapter 有区别吗？

```
原理完全一样，都是 比例缩放公式。
flutter_screenutil 封装更好，内置扩展方法、生命周期管理、横竖屏监听。
```

### 2. sp 和 dp 有什么区别？

```
dp：密度无关像素，不随系统字体大小变化
sp：缩放像素，跟随系统字体大小设置
字体用 sp，布局用 dp / .w / .h
```

### 3. 大屏（平板）怎么适配？

```
不能只做等比缩放，要用 LayoutBuilder 切换布局：
  手机：单列 ListView
  平板：双列 GridView / Master-Detail 布局
```

### 4. 横竖屏切换怎么处理？

```
flutter_screenutil 自动监听屏幕变化。
自定义方案需要在 didChangeDependencies 中重新计算。
或者用 OrientationBuilder 区分方向。
```
