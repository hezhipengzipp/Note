# Dart 扩展方法（Extension Methods）

---

## 一、是什么

**在不修改原类代码、不继承的情况下，给现有类添加方法**。和 Kotlin 的扩展函数是同一个概念。

```dart
// 给 String 加一个方法
extension StringExt on String {
  bool get isEmail => contains('@') && contains('.');
}

// 直接调用，就像 String 本来就有这个方法
print('test@gmail.com'.isEmail);  // true
print('hello'.isEmail);           // false
```

---

## 二、基本语法

```dart
extension 扩展名 on 目标类型 {
  // 方法
  // getter / setter
  // 运算符
  // 静态方法（Dart 3.3+）
}
```

```dart
// ① 实例方法
extension IntExt on int {
  int square() => this * this;
  bool get isEven => this % 2 == 0;
}

print(5.square());   // 25
print(4.isEven);     // true


// ② getter / setter
extension DateTimeExt on DateTime {
  String get formatted => '$year-$month-$day';
  bool get isWeekend => weekday == 6 || weekday == 7;
}

print(DateTime.now().formatted);   // 2026-4-16
print(DateTime.now().isWeekend);   // false


// ③ 运算符重载
extension ListExt<T> on List<T> {
  List<T> operator *(int times) {
    return List.generate(length * times, (i) => this[i % length]);
  }
}

print([1, 2] * 3);  // [1, 2, 1, 2, 1, 2]
```

## 三、泛型扩展

```dart
// 给所有 List<T> 加方法
extension ListExt<T> on List<T> {
  T? get firstOrNull => isEmpty ? null : first;
  T? get lastOrNull => isEmpty ? null : last;

  List<T> sortedBy(Comparable Function(T) keyOf) {
    return [...this]..sort((a, b) => keyOf(a).compareTo(keyOf(b)));
  }
}

var users = [User('张三', 30), User('李四', 20)];
var sorted = users.sortedBy((u) => u.age);
// [李四(20), 张三(30)]

var empty = <String>[];
print(empty.firstOrNull);  // null（不会抛异常）


// 给特定泛型类型加方法
extension NumListExt on List<num> {
  num get sum => fold(0, (a, b) => a + b);
  double get average => sum / length;
}

print([1, 2, 3, 4].sum);      // 10
print([1, 2, 3, 4].average);  // 2.5
// print(['a', 'b'].sum);      // ❌ 编译报错，List<String> 没有 sum
```

## 四、在 Flutter 中的实用扩展

### 4.1 BuildContext 扩展（最常用）

```dart
extension ContextExt on BuildContext {
  // 快捷访问 Theme
  ThemeData get theme => Theme.of(this);
  TextTheme get textTheme => Theme.of(this).textTheme;
  ColorScheme get colorScheme => Theme.of(this).colorScheme;

  // 快捷访问屏幕尺寸
  double get screenWidth => MediaQuery.of(this).size.width;
  double get screenHeight => MediaQuery.of(this).size.height;
  EdgeInsets get padding => MediaQuery.of(this).padding;

  // 快捷导航
  void pushPage(Widget page) {
    Navigator.of(this).push(MaterialPageRoute(builder: (_) => page));
  }
  void pop<T>([T? result]) => Navigator.of(this).pop(result);

  // 快捷 SnackBar
  void showSnack(String message) {
    ScaffoldMessenger.of(this).showSnackBar(
      SnackBar(content: Text(message)),
    );
  }
}

// 使用：
class MyPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text(
      'Hello',
      style: TextStyle(
        color: context.colorScheme.primary,     // 以前：Theme.of(context).colorScheme.primary
        fontSize: context.screenWidth * 0.05,   // 以前：MediaQuery.of(context).size.width * 0.05
      ),
    );
  }
}
```

### 4.2 Widget 扩展

```dart
extension WidgetExt on Widget {
  // 快捷包裹 Padding
  Widget padding(EdgeInsets padding) => Padding(padding: padding, child: this);
  Widget paddingAll(double value) => Padding(padding: EdgeInsets.all(value), child: this);

  // 快捷包裹 Center
  Widget get centered => Center(child: this);

  // 快捷包裹 GestureDetector
  Widget onTap(VoidCallback onTap) => GestureDetector(onTap: onTap, child: this);

  // 快捷包裹 Expanded
  Widget expanded([int flex = 1]) => Expanded(flex: flex, child: this);
}

// 使用：链式调用，减少嵌套
Text('Hello')
    .paddingAll(16)
    .centered
    .onTap(() => print('clicked'));

// 等价于以前的嵌套写法：
GestureDetector(
  onTap: () => print('clicked'),
  child: Center(
    child: Padding(
      padding: EdgeInsets.all(16),
      child: Text('Hello'),
    ),
  ),
)
```

### 4.3 String 扩展

```dart
extension StringExt on String {
  bool get isEmail => RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(this);
  bool get isPhone => RegExp(r'^1[3-9]\d{9}$').hasMatch(this);
  bool get isBlank => trim().isEmpty;
  bool get isNotBlank => !isBlank;

  // 安全转数字
  int? toIntOrNull() => int.tryParse(this);
  double? toDoubleOrNull() => double.tryParse(this);

  // 截断
  String truncate(int maxLength, {String suffix = '...'}) {
    return length <= maxLength ? this : '${substring(0, maxLength)}$suffix';
  }
}

print('test@gmail.com'.isEmail);  // true
print('13800138000'.isPhone);     // true
print('  '.isBlank);              // true
print('Hello World'.truncate(5)); // Hello...
```

### 4.4 DateTime 扩展

```dart
extension DateTimeExt on DateTime {
  String get ymd => '$year-${_pad(month)}-${_pad(day)}';
  String get hms => '${_pad(hour)}:${_pad(minute)}:${_pad(second)}';
  String get full => '$ymd $hms';

  String _pad(int n) => n.toString().padLeft(2, '0');

  bool get isToday {
    final now = DateTime.now();
    return year == now.year && month == now.month && day == now.day;
  }

  String get timeAgo {
    final diff = DateTime.now().difference(this);
    if (diff.inDays > 365) return '${diff.inDays ~/ 365}年前';
    if (diff.inDays > 30) return '${diff.inDays ~/ 30}个月前';
    if (diff.inDays > 0) return '${diff.inDays}天前';
    if (diff.inHours > 0) return '${diff.inHours}小时前';
    if (diff.inMinutes > 0) return '${diff.inMinutes}分钟前';
    return '刚刚';
  }
}

print(DateTime.now().full);        // 2026-04-16 14:30:00
print(DateTime.now().isToday);     // true
print(DateTime(2026, 4, 15).timeAgo);  // 1天前
```

## 五、冲突解决

```dart
// 两个扩展给同一个类型加了同名方法
extension A on String {
  String get hello => 'Hello from A';
}

extension B on String {
  String get hello => 'Hello from B';
}

// 直接调用 → 编译报错！
// print('test'.hello);  // ❌ 歧义，不知道用哪个

// 解决：显式指定用哪个扩展
print(A('test').hello);  // Hello from A
print(B('test').hello);  // Hello from B
```

```
作用域规则：

  如果当前文件 import 了两个扩展有同名方法 → 编译报错
  解决方式：
    ① 显式调用 ExtensionName(obj).method()
    ② 用 show/hide 控制 import
       import 'ext_a.dart' show A;  // 只导入 A
       import 'ext_b.dart' hide B;  // 隐藏 B
```

## 六、和 Kotlin 扩展函数对比

```kotlin
// Kotlin
fun String.isEmail(): Boolean = contains("@") && contains(".")
"test@gmail.com".isEmail()
```

```dart
// Dart
extension on String {
  bool get isEmail => contains('@') && contains('.');
}
'test@gmail.com'.isEmail
```

| | Dart Extension | Kotlin Extension |
|---|---|---|
| 语法 | `extension Name on Type { }` 块 | `fun Type.method()` 单个函数 |
| 组织方式 | 一个 extension 块里放多个方法 | 每个扩展函数独立声明 |
| 匿名扩展 | `extension on Type { }` 可以不命名 | 不支持 |
| 可见性控制 | 只能通过 import show/hide | `private fun Type.xxx()` |
| 运算符重载 | 支持 | 支持 |
| 静态方法 | Dart 3.3+ 支持 | 不支持（用伴生对象扩展） |
| 本质 | 都是静态分发，编译期确定，不是真正加方法到类上 | 同左 |

```
关键共同点：静态分发

  extension StringExt on String {
    void hello() => print('hello');
  }

  String? s = null;
  s.hello();  // ❌ Dart 编译报错（可空类型要处理）

  // 扩展方法在编译时确定，不是动态派发
  // 所以无法通过扩展方法实现多态
```

## 七、使用原则

```
适合用扩展方法：
  ✅ 给系统类加便捷方法（String、DateTime、BuildContext）
  ✅ 减少工具类静态方法（StringUtils.isEmail(s) → s.isEmail）
  ✅ 链式调用减少 Widget 嵌套
  ✅ 项目内统一的格式化、转换逻辑

不适合用：
  ❌ 替代正常的类方法（业务逻辑应该在类内部）
  ❌ 给扩展方法加太多状态依赖（扩展方法访问不到私有字段）
  ❌ 过度使用导致"到处都是魔法方法"，新人看不懂
```

## 八、面试回答模板

**Q: Dart 的扩展方法是什么？和 Kotlin 有什么区别？**

> Dart 用 `extension on Type` 语法给现有类添加方法，不需要修改原类或继承。和 Kotlin 扩展函数的原理一样，都是静态分发——编译期确定调用哪个方法，不是真的在类上加了方法。语法上的区别是 Dart 把多个扩展方法组织在一个 extension 块里，Kotlin 每个扩展函数独立声明。Flutter 中最常见的用法是扩展 BuildContext 简化 Theme、MediaQuery 的访问，以及扩展 Widget 实现链式调用减少嵌套。两个扩展有同名方法时 Dart 会编译报错，需要显式指定用哪个扩展，或者用 import show/hide 控制作用域。
