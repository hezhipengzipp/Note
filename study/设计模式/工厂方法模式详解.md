# 工厂方法模式详解

## 核心思想

```
简单工厂：一个工厂生产所有产品，新增产品要改工厂类   → 违反开闭原则
工厂方法：每种产品对应一个工厂，新增产品新增工厂类   → 符合开闭原则

核心：把"创建对象"的决定推迟到子类
```

---

## 类结构

```
        «interface»
        Product
            ▲
            │ implements
    ┌───────┴────────┐
ConcreteProductA  ConcreteProductB

        «abstract»
        Creator
        + createProduct(): Product   ← 抽象工厂方法
        + doSomething()              ← 模板方法（调用工厂方法）
            ▲
            │ extends
    ┌───────┴────────┐
CreatorA           CreatorB
+ createProduct()  + createProduct()
  → new ProductA     → new ProductB
```

## 代码示例

```java
// 产品接口
public interface Button {
    void render();
    void onClick();
}

// 具体产品 A
public class WindowsButton implements Button {
    @Override
    public void render() { System.out.println("渲染 Windows 风格按钮"); }
    @Override
    public void onClick() { System.out.println("Windows 按钮点击"); }
}

// 具体产品 B
public class MacButton implements Button {
    @Override
    public void render() { System.out.println("渲染 Mac 风格按钮"); }
    @Override
    public void onClick() { System.out.println("Mac 按钮点击"); }
}

// 抽象工厂：声明工厂方法
public abstract class Dialog {

    // 工厂方法：子类决定创建哪种产品
    public abstract Button createButton();

    // 模板方法：使用工厂方法
    public void renderDialog() {
        Button button = createButton();   // 调用工厂方法
        button.render();
        button.onClick();
    }
}

// 具体工厂 A
public class WindowsDialog extends Dialog {
    @Override
    public Button createButton() {
        return new WindowsButton();
    }
}

// 具体工厂 B
public class MacDialog extends Dialog {
    @Override
    public Button createButton() {
        return new MacButton();
    }
}

// 客户端
public class Application {
    private Dialog dialog;

    public void initialize(String os) {
        if (os.equals("Windows")) {
            dialog = new WindowsDialog();
        } else {
            dialog = new MacDialog();
        }
    }

    public void render() {
        dialog.renderDialog();    // 不关心具体按钮类型
    }
}
```

## 新增产品只需扩展，不修改

```java
// 新增 Linux 风格，只需新增两个类，原有代码完全不动

public class LinuxButton implements Button {
    @Override
    public void render() { System.out.println("渲染 Linux 风格按钮"); }
    @Override
    public void onClick() { System.out.println("Linux 按钮点击"); }
}

public class LinuxDialog extends Dialog {
    @Override
    public Button createButton() {
        return new LinuxButton();
    }
}
// Dialog、Button 接口、已有产品类完全不动 ✅ 符合开闭原则
```

## 与简单工厂对比

```java
// 简单工厂（不是设计模式，只是一种写法）
public class ButtonFactory {
    public static Button create(String type) {
        switch (type) {
            case "Windows": return new WindowsButton();
            case "Mac":     return new MacButton();
            // 新增 Linux → 必须修改这个 switch ❌ 违反开闭原则
            default: throw new IllegalArgumentException();
        }
    }
}

// 工厂方法
// 新增产品 → 新增子类，不改已有代码 ✅ 符合开闭原则
```

| | 简单工厂 | 工厂方法 |
|---|---|---|
| 工厂数量 | 1 个 | N 个（每种产品一个）|
| 新增产品 | 改工厂类 | 加新工厂类 |
| 开闭原则 | 违反 | 遵守 |
| 复杂度 | 低 | 稍高 |
| 适合场景 | 产品种类固定 | 产品种类会扩展 |

public class UserFragment extends Fragment {
    public static UserFragment newInstance(int userId) {
        UserFragment fragment = new UserFragment();

        args.putInt("userId", userId);
        fragment.setArguments(args);

    }

## 总结

```
工厂方法模式解决的问题：
  创建对象时不想指定具体类，把"创建哪种对象"的决定留给子类

三个角色：
  Product（产品接口）        → 定义产品行为
  Creator（抽象工厂）        → 声明抽象工厂方法
  ConcreteCreator（具体工厂）→ 实现工厂方法，返回具体产品

优点：
  ✅ 符合开闭原则，新增产品不改已有代码
  ✅ 单一职责，每个工厂只负责一种产品
  ✅ 解耦客户端与具体产品类

缺点：
  ❌ 每新增一种产品就要新增一个工厂类，类数量增多
  ❌ 只支持同一类产品，多类产品需要用抽象工厂模式
```
