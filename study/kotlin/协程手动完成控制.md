# Kotlin 协程手动完成控制指南

## 概述

Kotlin 协程提供了多种方式来手动控制异步操作的完成，类似于 Dart 的 `Completer`。主要工具包括：

- `CompletableDeferred` - 可手动完成的 Deferred
- `suspendCoroutine` / `suspendCancellableCoroutine` - 将回调转为挂起函数
- `Channel` - 协程间通信

## Kotlin vs Dart 对照表

| Dart | Kotlin | 说明 |
|------|--------|------|
| `Completer<T>` | `CompletableDeferred<T>` | 可手动完成的异步值 |
| `completer.future` | `deferred.await()` | 等待结果 |
| `completer.complete(value)` | `deferred.complete(value)` | 成功完成 |
| `completer.completeError(error)` | `deferred.completeError(error)` | 以错误完成 |
| `completer.isCompleted` | `deferred.isCompleted` | 检查是否已完成 |
| `Future<T>` | `Deferred<T>` / `suspend fun` | 异步操作 |

## 何时需要手动控制完成？

| 场景 | 是否需要 | Kotlin 推荐方案 |
|------|---------|----------------|
| 普通的协程函数 | ❌ 不需要 | 直接使用 `suspend fun` |
| 将回调 API 转为协程 | ✅ 需要 | `suspendCancellableCoroutine` |
| 等待外部事件 | ✅ 需要 | `CompletableDeferred` 或 `Channel` |
| 创建可取消的异步操作 | ✅ 需要 | `suspendCancellableCoroutine` |
| 单元测试中控制完成时机 | ✅ 需要 | `CompletableDeferred` |
| 协程间通信 | ✅ 需要 | `Channel` 或 `Flow` |

## 1. CompletableDeferred 基础用法

`CompletableDeferred` 是 `Deferred` 的一个实现，允许手动完成。

```kotlin
import kotlinx.coroutines.*

fun main() = runBlocking {
    val deferred = CompletableDeferred<String>()
    
    // 在另一个协程中完成它
    launch {
        delay(1000)
        deferred.complete("完成了！")
    }
    
    // 等待结果
    val result = deferred.await()
    println(result) // 输出: 完成了！
}
```

### 基本方法

```kotlin
val deferred = CompletableDeferred<String>()

// 完成
deferred.complete("成功")

// 以异常完成
deferred.completeExceptionally(Exception("失败"))

// 检查是否已完成
if (!deferred.isCompleted) {
    deferred.complete("延迟完成")
}

// 等待结果
val result = deferred.await()

// 取消
deferred.cancel()
```

## 2. 将回调 API 转为协程

### 使用 suspendCoroutine（不可取消）

```kotlin
import kotlin.coroutines.resume
import kotlin.coroutines.resumeWithException
import kotlin.coroutines.suspendCoroutine

class CallbackAPI {
    fun fetchData(
        onSuccess: (String) -> Unit,
        onError: (Exception) -> Unit
    ) {
        // 模拟异步回调
        Thread {
            Thread.sleep(1000)
            if (System.currentTimeMillis() % 2 == 0L) {
                onSuccess("数据加载成功")
            } else {
                onError(Exception("加载失败"))
            }
        }.start()
    }
}

suspend fun fetchDataAsCoroutine(): String = suspendCoroutine { continuation ->
    val api = CallbackAPI()
    api.fetchData(
        onSuccess = { data -> continuation.resume(data) },
        onError = { error -> continuation.resumeWithException(error) }
    )
}

fun main() = runBlocking {
    try {
        val result = fetchDataAsCoroutine()
        println(result)
    } catch (e: Exception) {
        println("错误: ${e.message}")
    }
}
```

### 使用 suspendCancellableCoroutine（推荐）

支持取消操作，更适合实际场景。

```kotlin
import kotlinx.coroutines.*
import kotlin.coroutines.resume
import kotlin.coroutines.resumeWithException

class NetworkAPI {
    fun request(
        url: String,
        onSuccess: (String) -> Unit,
        onError: (Exception) -> Unit
    ): CancellableRequest {
        val request = CancellableRequest()
        Thread {
            var i = 0
            while (i < 10 && !request.isCancelled) {
                Thread.sleep(100)
                i++
            }
            if (!request.isCancelled) {
                onSuccess("来自 $url 的数据")
            }
        }.start()
        return request
    }
    
    class CancellableRequest {
        var isCancelled = false
        fun cancel() { isCancelled = true }
    }
}

suspend fun fetchWithCancellation(url: String): String = 
    suspendCancellableCoroutine { continuation ->
        val api = NetworkAPI()
        val request = api.request(
            url,
            onSuccess = { data -> continuation.resume(data) },
            onError = { error -> continuation.resumeWithException(error) }
        )
        
        // 设置取消处理
        continuation.invokeOnCancellation {
            request.cancel()
        }
    }

fun main() = runBlocking {
    val job = launch {
        try {
            val result = fetchWithCancellation("https://api.example.com")
            println(result)
        } catch (e: CancellationException) {
            println("请求被取消")
        }
    }
    
    delay(300)
    job.cancel() // 取消请求
    println("已发送取消信号")
}
```

## 3. 等待外部事件

### 使用 CompletableDeferred

```kotlin
import kotlinx.coroutines.*
import androidx.compose.runtime.*
import androidx.compose.material3.*

class UserInputHandler {
    private var deferred: CompletableDeferred<String>? = null
    
    suspend fun waitForUserInput(): String {
        deferred = CompletableDeferred()
        return deferred!!.await()
    }
    
    fun onButtonClicked(value: String) {
        deferred?.complete(value)
    }
    
    fun cancel() {
        deferred?.cancel()
    }
}

@Composable
fun WaitForUserChoice() {
    val handler = remember { UserInputHandler() }
    val scope = rememberCoroutineScope()
    
    Column {
        Button(onClick = { 
            scope.launch {
                handler.onButtonClicked("选项 A")
            }
        }) {
            Text("选项 A")
        }
        
        Button(onClick = { 
            scope.launch {
                handler.onButtonClicked("选项 B")
            }
        }) {
            Text("选项 B")
        }
    }
    
    LaunchedEffect(Unit) {
        val choice = handler.waitForUserInput()
        println("用户选择了: $choice")
    }
}
```

### 使用 Channel（更灵活）

`Channel` 适合需要多次传递值的场景。

```kotlin
import kotlinx.coroutines.*
import kotlinx.coroutines.channels.Channel

class EventHandler {
    private val channel = Channel<String>()
    
    suspend fun waitForEvent(): String {
        return channel.receive()
    }
    
    suspend fun sendEvent(event: String) {
        channel.send(event)
    }
    
    fun close() {
        channel.close()
    }
}

fun main() = runBlocking {
    val handler = EventHandler()
    
    // 生产者协程
    launch {
        repeat(3) { i ->
            delay(1000)
            handler.sendEvent("事件 $i")
        }
        handler.close()
    }
    
    // 消费者协程
    launch {
        repeat(3) {
            val event = handler.waitForEvent()
            println("接收到: $event")
        }
    }
}
```

## 4. 创建可取消的异步操作

```kotlin
import kotlinx.coroutines.*

class CancellableOperation {
    private var deferred: CompletableDeferred<String>? = null
    private var job: Job? = null
    
    fun start(): Deferred<String> {
        deferred = CompletableDeferred()
        
        job = CoroutineScope(Dispatchers.Default).launch {
            delay(5000)
            if (isActive && deferred?.isCompleted == false) {
                deferred?.complete("操作完成")
            }
        }
        
        return deferred!!
    }
    
    fun cancel() {
        job?.cancel()
        deferred?.completeExceptionally(CancellationException("操作被取消"))
    }
}

fun main() = runBlocking {
    val operation = CancellableOperation()
    val deferred = operation.start()
    
    // 2秒后取消
    launch {
        delay(2000)
        println("取消操作...")
        operation.cancel()
    }
    
    try {
        val result = deferred.await()
        println(result)
    } catch (e: CancellationException) {
        println("错误: ${e.message}") // 输出: 错误: 操作被取消
    }
}
```

## 5. 单元测试中控制完成时机

```kotlin
import kotlinx.coroutines.*
import kotlinx.coroutines.test.*
import org.junit.Test
import kotlin.test.assertEquals

interface DataService {
    suspend fun fetchData(): String
}

class RealDataService : DataService {
    override suspend fun fetchData(): String {
        delay(1000)
        return "real data"
    }
}

class MockDataService : DataService {
    private var deferred: CompletableDeferred<String>? = null
    
    override suspend fun fetchData(): String {
        deferred = CompletableDeferred()
        return deferred!!.await()
    }
    
    fun completeWithData(data: String) {
        deferred?.complete(data)
    }
    
    fun completeWithError(error: Throwable) {
        deferred?.completeExceptionally(error)
    }
}

class DataServiceTest {
    @Test
    fun `测试数据加载`() = runTest {
        val mockService = MockDataService()
        
        // 启动异步操作
        val job = launch {
            val result = mockService.fetchData()
            assertEquals("test data", result)
        }
        
        // 此时协程被挂起，等待数据
        
        // 手动完成
        mockService.completeWithData("test data")
        
        // 等待协程完成
        job.join()
    }
    
    @Test
    fun `测试错误处理`() = runTest {
        val mockService = MockDataService()
        
        val job = launch {
            try {
                mockService.fetchData()
                assert(false) { "不应该执行到这里" }
            } catch (e: Exception) {
                assertEquals("test error", e.message)
            }
        }
        
        mockService.completeWithError(Exception("test error"))
        job.join()
    }
}
```

## 6. 超时处理

Kotlin 协程提供了内置的超时支持。

```kotlin
import kotlinx.coroutines.*

suspend fun fetchWithTimeout(url: String): String = withTimeout(5000) {
    // 模拟网络请求
    delay(3000)
    "数据来自 $url"
}

// 不抛异常的版本
suspend fun fetchWithTimeoutOrNull(url: String): String? = withTimeoutOrNull(5000) {
    delay(3000)
    "数据来自 $url"
}

fun main() = runBlocking {
    // 会抛出 TimeoutCancellationException
    try {
        val result = withTimeout(1000) {
            delay(2000)
            "完成"
        }
    } catch (e: TimeoutCancellationException) {
        println("超时了！")
    }
    
    // 返回 null 而不是抛异常
    val result = withTimeoutOrNull(1000) {
        delay(2000)
        "完成"
    }
    println(result) // 输出: null
}
```

## 7. 多个异步操作的协调

### 等待多个操作完成

```kotlin
import kotlinx.coroutines.*

fun main() = runBlocking {
    val deferred1 = CompletableDeferred<String>()
    val deferred2 = CompletableDeferred<Int>()
    val deferred3 = CompletableDeferred<Boolean>()
    
    // 在不同的协程中完成它们
    launch {
        delay(100)
        deferred1.complete("完成1")
    }
    
    launch {
        delay(200)
        deferred2.complete(42)
    }
    
    launch {
        delay(150)
        deferred3.complete(true)
    }
    
    // 等待所有完成
    val results = awaitAll(deferred1, deferred2, deferred3)
    println(results) // [完成1, 42, true]
}
```

### select 表达式（等待第一个完成）

```kotlin
import kotlinx.coroutines.*
import kotlinx.coroutines.selects.select

fun main() = runBlocking {
    val deferred1 = async {
        delay(1000)
        "结果1"
    }
    
    val deferred2 = async {
        delay(500)
        "结果2"
    }
    
    val firstResult = select<String> {
        deferred1.onAwait { it }
        deferred2.onAwait { it }
    }
    
    println("最快的结果: $firstResult") // 输出: 最快的结果: 结果2
}
```

## 8. 高级示例：实现重试机制

```kotlin
import kotlinx.coroutines.*
import kotlin.time.Duration
import kotlin.time.Duration.Companion.seconds

suspend fun <T> retryWithExponentialBackoff(
    maxRetries: Int = 3,
    initialDelay: Duration = 1.seconds,
    maxDelay: Duration = 10.seconds,
    factor: Double = 2.0,
    block: suspend () -> T
): T {
    var currentDelay = initialDelay
    repeat(maxRetries - 1) { attempt ->
        try {
            return block()
        } catch (e: Exception) {
            println("尝试 ${attempt + 1} 失败: ${e.message}")
            delay(currentDelay)
            currentDelay = (currentDelay * factor).coerceAtMost(maxDelay)
        }
    }
    return block() // 最后一次尝试
}

fun main() = runBlocking {
    var attempts = 0
    val result = retryWithExponentialBackoff {
        attempts++
        if (attempts < 3) {
            throw Exception("临时错误")
        }
        "成功！"
    }
    println(result)
}
```

## 9. Flow：响应式数据流

对于需要多次发送值的场景，`Flow` 比 `CompletableDeferred` 更合适。

```kotlin
import kotlinx.coroutines.*
import kotlinx.coroutines.flow.*

class DataStreamHandler {
    private val _dataFlow = MutableSharedFlow<String>()
    val dataFlow: SharedFlow<String> = _dataFlow
    
    suspend fun emitData(data: String) {
        _dataFlow.emit(data)
    }
}

fun main() = runBlocking {
    val handler = DataStreamHandler()
    
    // 消费者
    launch {
        handler.dataFlow.collect { data ->
            println("接收到: $data")
        }
    }
    
    // 生产者
    launch {
        repeat(5) { i ->
            delay(500)
            handler.emitData("数据 $i")
        }
    }
    
    delay(3000)
}
```

## 注意事项和最佳实践

### 1. 只能完成一次

```kotlin
val deferred = CompletableDeferred<String>()
deferred.complete("第一次")
// deferred.complete("第二次") // 会抛出 IllegalStateException

// 安全的做法
if (!deferred.isCompleted) {
    deferred.complete("安全完成")
}
```

### 2. 协程取消

协程取消时，未完成的 `CompletableDeferred` 也会被取消。

```kotlin
val job = CoroutineScope(Dispatchers.Default).launch {
    val deferred = CompletableDeferred<String>()
    launch {
        delay(2000)
        deferred.complete("完成")
    }
    deferred.await() // 如果 job 被取消，这里会抛出 CancellationException
}

delay(500)
job.cancel()
```

### 3. 结构化并发

优先使用结构化并发，而不是全局协程。

```kotlin
// ❌ 不好的做法
GlobalScope.launch {
    val deferred = CompletableDeferred<String>()
    // ...
}

// ✅ 好的做法
suspend fun doWork() = coroutineScope {
    val deferred = CompletableDeferred<String>()
    launch {
        // ...
    }
    deferred.await()
}
```

### 4. 异常处理

```kotlin
val deferred = CompletableDeferred<String>()

try {
    // 某些操作
    throw Exception("出错了")
} catch (e: Exception) {
    deferred.completeExceptionally(e)
}

// 使用时
try {
    val result = deferred.await()
} catch (e: Exception) {
    println("捕获异常: ${e.message}")
}
```

## 完整示例：网络请求封装

```kotlin
import kotlinx.coroutines.*
import java.net.HttpURLConnection
import java.net.URL
import kotlin.coroutines.resume
import kotlin.coroutines.resumeWithException

class NetworkClient {
    suspend fun get(url: String, timeoutMs: Long = 5000): String = 
        withTimeout(timeoutMs) {
            suspendCancellableCoroutine { continuation ->
                val thread = Thread {
                    try {
                        val connection = URL(url).openConnection() as HttpURLConnection
                        connection.requestMethod = "GET"
                        connection.connectTimeout = timeoutMs.toInt()
                        connection.readTimeout = timeoutMs.toInt()
                        
                        val response = connection.inputStream.bufferedReader().use { it.readText() }
                        continuation.resume(response)
                    } catch (e: Exception) {
                        continuation.resumeWithException(e)
                    }
                }
                
                thread.start()
                
                continuation.invokeOnCancellation {
                    thread.interrupt()
                }
            }
        }
}

fun main() = runBlocking {
    val client = NetworkClient()
    
    try {
        val response = client.get("https://api.github.com", timeoutMs = 3000)
        println("响应长度: ${response.length}")
    } catch (e: TimeoutCancellationException) {
        println("请求超时")
    } catch (e: Exception) {
        println("请求失败: ${e.message}")
    }
}
```

## 总结

| 场景 | Dart | Kotlin |
|------|------|--------|
| 简单异步 | `async/await` | `suspend fun` |
| 回调转异步 | `Completer` | `suspendCancellableCoroutine` |
| 手动控制完成 | `Completer` | `CompletableDeferred` |
| 多次发送值 | `Stream` | `Channel` / `Flow` |
| 取消支持 | 手动实现 | 内置支持 |
| 超时处理 | 手动实现 | `withTimeout` |

### 选择指南

- **简单异步**: 使用 `suspend fun` 和 `async`
- **回调 API**: 使用 `suspendCancellableCoroutine`
- **手动控制时机**: 使用 `CompletableDeferred`
- **协程间通信**: 使用 `Channel` 或 `Flow`
- **需要取消**: 优先使用 `suspendCancellableCoroutine` 和协程作用域

Kotlin 协程的优势在于更好的取消支持、结构化并发和丰富的内置工具函数。
